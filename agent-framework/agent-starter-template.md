# Agent Starter Template

> Use this template to build any specialized AI agent for your business.
> Fill in each section. The more specific you are, the sharper the output.

---

## 1. Identity

**Agent Name:** [Give it a name — helps with mental model]
**Role:** [e.g., Head of Sales, Brand Director, Analytics Lead]
**Voice:** [e.g., "Direct, warm, data-backed" or "Sharp, pattern-oriented, action-focused"]

Describe the personality in 2-3 sentences. How does this agent communicate? What's its attitude toward the work?

---

## 2. Context

What does this agent need to know about your business?

- **Your business:** [What you do, who you serve]
- **Your products/services:** [What you sell or offer]
- **Your proof points:** [Specific metrics, results, case studies — numbers only]
- **Your tech stack:** [Tools you use]
- **Your brand voice:** [How you sound across platforms]

The more real data you feed here, the less generic the output becomes.

---

## 3. Processes

Define step-by-step workflows for each task this agent handles.

### Process 1: [Name]
When asked to [trigger]:
1. [Step 1]
2. [Step 2]
3. [Step 3]
4. [Output format]

### Process 2: [Name]
When asked to [trigger]:
1. [Step 1]
2. [Step 2]
3. [Step 3]
4. [Output format]

Add as many processes as needed. Each should be specific enough that someone unfamiliar with your business could follow them.

---

## 4. Deliverables

What does this agent produce? Define the format for every output:

- [Output type 1]: [Format and requirements]
- [Output type 2]: [Format and requirements]

Every output should include:
- The actual deliverable (ready to use, not a draft with placeholders)
- A brief strategy note explaining WHY this approach
- Suggested timing or next steps

---

## 5. Anti-Patterns

What should this agent NEVER do? This is the most important section for quality control.

- Never [specific bad behavior]
- Never [specific bad behavior]
- Never [specific bad behavior]

Anti-patterns prevent hallucinations, off-brand messaging, and errors that erode trust. Be specific — "never be generic" is too vague. "Never write outreach without referencing a specific proof point" is actionable.

---

## Why This Structure Works

Traditional AI prompts: "Act as a sales agent and write me an email."
→ Generic output. No context. No guardrails.

Agent with this framework:
- **Identity** gives it a consistent voice across every interaction
- **Context** feeds it YOUR data so it stops hallucinating
- **Processes** replace vague instructions with repeatable workflows
- **Deliverables** define what "done" looks like
- **Anti-patterns** prevent the mistakes you've already learned from

The result: an AI that produces the same quality whether you use it at 9 AM or 11 PM, on Monday or Friday, for the first time or the hundredth.

---

## Getting Started

1. Copy this template
2. Fill in ONE agent (start with your most repeated task)
3. Save as a markdown file
4. Use it as a system prompt or Claude Skill
5. Test it for a week — note what works and what doesn't
6. Add anti-patterns for every mistake it makes
7. Repeat until the output is consistently good
8. Build the next agent

One agent at a time. That's how systems get built.
